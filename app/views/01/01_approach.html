Creating an account

On the surface, this should be an easy thing to do - all you need to create an account are a few input fields right?

Technically accurate - but when reading the design system docs, it gets a bit more involving...

First you need to consider the Create accounts pattern - that's what is being utilised here.

Create accounts

The first factor to consider is if you really need your users to creat an account. Do they NEED to have one to be able to use your service?

In the case of the Civil Service Jobs site yes, the users do need an account. There is a lot of personal information that can be added and managed when applying for jobs, so the account allows the user to maintain their control over that.

The Dribbble shot that I had in mind when I first started this project seemed to be for a Politcal Newsletter.

I'd argue that the user doesn't need an account to get this - they just need to provide an email address.
